# `develop` 자동 백업 및 무결성 병합 (Sync Develop)

이 강력한 내부 스킬(워크플로우)은 현재 작업 중인 브랜치에서 `develop` 브랜치의 최신 코드를 안전하게 당겨오고, 그 과정에서 발생하는 지옥 같은 'Merge Conflict(충돌)'이나 '바뀐 남의 코드로 인한 컴파일 에러'를 AI가 알아서 우아하게 병합(통합)해주는 워크플로우입니다.

어느 브랜치에서든 터미널 통신창에 `/sync_develop` 이라고 치면 아래 단계를 알아서 수행합니다.

// turbo
1. `git fetch origin develop` 으로 원격지의 변경 사항만 일단 눈치껏 가져옵니다.

// turbo
2. `git merge origin/develop` 커맨드로 실질적인 코드 합치기를 시도합니다.

3. 🚨 **만약 이 과정에서 끔찍한 Merge Conflict(코드 충돌) 파일들이 발생했다면?**
   - 좌절하지 말고 충돌 메시지를 읽은 뒤, 충돌이 발생한 파일들을 에디터로 직접 스캔합니다.
   - `<<<<<<< HEAD` 부터 `======` `>>>>>>> develop` 로 이어지는 마커를 찾아, 양쪽 개발자의 코드 문맥과 의도(주로 Spring Controller, Service 패턴)에 맞게 **두 코드를 가장 유기적으로 합칩니다!**
   - 합친 파일의 찌꺼기 괄호를 없앤 뒤 터미널에서 `git add .` 와 `git commit -m "Auto-resolved merge conflicts with develop"` 을 실행하여 병합을 끝냅니다.

// turbo
4. ✅ 통합/병합이 1차로 무사히 끝났어도, 남이 수정한 함수/인터페이스 이름과 내 코드가 충돌할 수 있으니 터미널에서 `./gradlew clean build -x test` 를 돌려서 컴파일 검열을 시작합니다.

5. 검열에서 자바 에러가 터졌다면?
   - 터미널의 에러 로그(Line)를 읽고 내 파일의 코드를 직접 수정하여 빨간 줄을 모조리 제거하고 다시 한 치의 에러 없이 빌드합니다!

6. 📝 **`CLAUDE.md` 지침서 스스로 업데이트 (자가 학습/진화)**
   - 통합이 완료된 뒤 `develop`에서 넘어온 새로운 코드들을 가볍게 스캔합니다.
   - 만약 '새로운 라이브러리/DB가 추가되었거나', '중대한 설계 패턴' 등이 감지되었다면 망설이지 말고 최상단의 `CLAUDE.md` 파일을 직접 열어서 **새로운 규칙이나 추가된 아키텍처를 영구적으로 항목화하여 기록(업데이트)** 하세요! (미래의 AI 에이전트 분신들이 똑똑해질 수 있도록 지식을 쌓아둡니다.)
